TOPIC 1: INTRODUCTION TO TCP
────────────────────────────────────────────────────────────

Definition:
A TCP is defined as a mechanism for managing shared resources in a concurrent environment.

Key Properties:
1. Correctness — must satisfy all TCP requirements
2. Scalability — performance under increasing load
3. Durability — behavior in failure scenarios

Example:
Consider a system with 7 processes/nodes. The TCP operates
as follows:
Step 1: Initialize the TCP with default parameters
Step 2: Apply UDP to determine priority/order
Step 3: Execute the operation while maintaining socket invariants
Step 4: Verify correctness using post-conditions

Important Note: UDP is preferred when memory is not a constraint and speed is critical.

────────────────────────────────────────────────────────────
TOPIC 3: ALGORITHMS & COMPLEXITY ANALYSIS
────────────────────────────────────────────────────────────

Algorithm — Tcp:
```
function TCP(input):
    if input is empty:
        return null
    
    result = initialize()
    for each element in input:
        process(element)
        update(result)
    
    return result
```

Time Complexity: O(1)
Space Complexity: O(log n)
Best Case: O(1)
Worst Case: O(n)

IMPORTANT FORMULAE
────────────────────────────────────────────────────────────

1. Efficiency = (Useful Work Done) / (Total Resources Used)
2. Throughput = (Number of Processes Completed) / (Time Period)  
3. Waiting Time = Completion Time - Burst Time - Arrival Time
4. Turnaround Time = Completion Time - Arrival Time
